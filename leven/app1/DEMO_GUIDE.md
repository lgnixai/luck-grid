# Grid Demo 使用指南

## 快速启动

### 方法一：使用启动脚本
```bash
cd leven/app1
./start.sh
```

### 方法二：手动启动
```bash
cd leven/app1
npm install
npm run dev
```

## 演示功能

### 1. 基础交互
- **点击单元格** - 选择单个单元格
- **双击单元格** - 查看单元格详情（控制台输出）
- **点击列标题** - 选择整列
- **右键点击** - 显示上下文菜单（控制台输出）

### 2. 键盘操作
- `Ctrl/Cmd + C` - 复制选中内容
- `Ctrl/Cmd + V` - 粘贴内容
- `Delete` - 删除选中内容
- `Ctrl/Cmd + Z` - 撤销操作
- `Ctrl/Cmd + Y` - 重做操作

### 3. 数据类型展示
- **姓名** - 文本类型
- **年龄** - 数字类型
- **邮箱** - 文本类型
- **部门** - 选择类型（多选）
- **是否在职** - 布尔类型
- **入职日期** - 文本类型
- **薪资** - 数字类型
- **操作** - 按钮类型（可点击）

### 4. 滚动功能
- 垂直滚动查看更多行
- 水平滚动查看更多列
- 冻结列功能（第一列固定）

## 技术特点

### 组件架构
- **Grid.tsx** - 核心网格渲染组件
- **GridView.tsx** - 业务逻辑和数据管理
- **类型系统** - 完整的 TypeScript 类型定义

### 性能优化
- 虚拟化渲染（只渲染可见区域）
- 高效的滚动处理
- 内存优化的数据管理

### 可扩展性
- 模块化的组件设计
- 灵活的主题配置
- 易于添加新的数据类型

## 自定义配置

### 修改数据
编辑 `src/data/mockData.ts` 文件来修改演示数据：

```typescript
// 添加新字段
{
  id: 'field_9',
  name: '新字段',
  type: 'text',
  description: '字段描述',
}

// 添加新记录
{
  id: 'record_9',
  fields: {
    field_1: '新用户',
    field_2: 30,
    // ... 其他字段
  },
}
```

### 修改主题
在 `src/components/GridView.tsx` 中修改主题配置：

```typescript
<Grid
  theme={{
    backgroundColor: '#ffffff',
    borderColor: '#e5e7eb',
    headerBackgroundColor: '#f9fafb',
    // ... 其他主题配置
  }}
/>
```

## 开发说明

这个 demo 展示了如何从复杂的 Teable GridViewBaseInner 组件中抽离出核心功能，创建一个独立、可复用的表格组件。

### 主要简化
1. 移除了复杂的业务逻辑（权限、协作、AI 功能等）
2. 简化了数据管理（使用静态模拟数据）
3. 保留了核心的表格交互功能
4. 提供了清晰的组件接口

### 扩展建议
1. 添加数据持久化
2. 实现真实的编辑功能
3. 添加更多数据类型支持
4. 实现列排序和筛选
5. 添加分组功能
